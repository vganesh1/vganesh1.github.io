SATGraf.jar

Instructions

To run the jar above, use the following command from the terminal: java -jar SATGraf.jar <path to input SAT formula> <tool> <community algorithm> <layout algorithm> <optional: number of conflicts>. Each of the options in the command above are explained bellow:

  • path to input SAT formula
    • Path to the desired SAT input formula which has to follow proper CNF format conventions.
  • tool options
    • community
      • Generate the graph with community structure of the input SAT formula.
    • evolution
      • Generate a series of graphs with community structure representing the evolution of the SAT formula as a SAT solver is solving it.
    • implication
      • Generate a graph where the nodes can be assigned a value and the graph will reflect this decision. (Currently does only uses the grid layout algorithm and does not use community algorithms. Work in progress.)
  • community algorithm options
    • cnm
      • Clauset-Newman-Moore community detection algorithm. When compared with 'ol', this will find higher quality communities but it is slower as a result.
    • ol
      • Online community detection algorithm. When compared with 'cnm', this will be much faster but does not guarantee communities of the same quality.
  • layout algorithm options
    • kk
      • A variation of the Kamada-Kawai force directed graph layout algorithm.
    • grid
      • A simple grid pattern layout which groups communities together. It is fast but does not reflect the community structure as well.
  • optional: number of conflicts
    • When using the evolution tool, you can specify after how many conflicts which the SAT solver encounters should SATGraf generate a graph. Example: if the number of conflicts is 2, a graph will be generated every 2 conflicts. If this option is not specified, it will be defaulted to 1.
